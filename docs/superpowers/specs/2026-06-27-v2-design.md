> 日期：2026-06-27

## 1. 总览

### 1.1 目标

将 clone 从"功能可用"提升为"体验领先"，在隐私保护、AI 智能、报告质量三个维度超越竞品。

### 1.2 Phase 划分（分阶段交付）

```
Phase 1: 隐私 + 记录增强（基础设施）
  ├─ 热力图时区 bug 修复 + 样式优化（2 行代码，立即修复）
  ├─ 截图分析后自动删除（可配置）
  ├─ 敏感场景硬跳过
  ├─ 三种记录方式共存（定时自动 / 全局快捷键 / 窗口内回车）
  ├─ 隐私三档设置
  ├─ 模板 clustering 字段向后兼容迁移
  └─ v1→v2 数据迁移逻辑（db.ts version check）

Phase 2: Today 页 + 计划系统 + 报告增强（核心体验）
  ├─ Today 页完整重写
  ├─ 今日计划列表（checkbox + 拖拽排序）
  ├─ 计划注入报告生成（计划 vs 实际对比）
  ├─ 报告模板增强（聚类策略 + 智能推荐）
  └─ AI 洞察建议（热力图/时长下方自动生成）

Phase 3: 设置页 + Agent + 帮助/订阅/邀请（完整产品）
  ├─ 设置页 5 卡片重写
  ├─ 内置 AI 对话（Agent 页）
  ├─ 订阅页（纯前端 UI）
  ├─ 邀请激励页（邀请码 + 双向奖励）
  └─ 帮助页（FAQ + 使用指南）
```

### 1.3 设计原则

- **所有弹窗改为页面内内联展示**（报告详情、模板编辑、设置配置、确认删除、截图预览等）
- **隐私优先**：默认开启敏感场景跳过，默认分析后删除截图
- **零门槛 AI**：内置对话，不依赖外部工具
- **智能推荐**：减少用户选择负担

## 4. Phase 1: 隐私 + 记录增强

### 4.1 截图即销毁

**流程**：

```
截图 → 保存到 screenshots/ → AI 分析 → 创建 workRecord
     → 如果 autoDeleteScreenshots=true → fs.unlinkSync(filePath)
     → updateScreenshot(id, { path: '' })  // 标记已销毁
```

**实现位置**：`electron/screenshot.ts` 的 `captureAndAnalyze()` 和 `captureNow()`

**前端反馈**：
- 截图查看时，path 为空显示"截图已按隐私策略销毁"
- Today Hero 区动态显示"截图分析后即销毁 ✓"

### 4.2 系统通知（桌面右上角 Toast）

每次截图分析完成后，在桌面右上角弹出通知提示。

**通知内容**：

| 场景 | 通知文案 |
|------|---------|
| 定时自动记录成功 | `✅ 已记录：{category} · {summary 前 20 字}` |
| 快捷键手动记录成功 | `✅ 快速记录：{category} · {summary 前 20 字}` |
| 窗口内回车记录成功 | `✅ 已记录：{category} · {summary 前 20 字}` |
| 敏感场景跳过 | `⏭️ 已跳过敏感内容` |
| 记录失败 | `❌ 记录失败：{error 前 30 字}` |

**实现**：

- 使用 Electron `Notification` API（主进程）
- 通知位置：系统默认（Windows 右下角，macOS 右上角）
- 通知持续：3 秒后自动消失
- 窗口聚焦时不弹出（避免干扰正在看 app 的用户）
- 设置项：`Settings.showNotifications: boolean`（默认 true）
- 在"截图与记录"卡片中加"完成通知"开关

**触发位置**：`electron/screenshot.ts` 的 `captureAndAnalyze()` 和 `captureNow()` 完成后

### 4.2 敏感场景硬跳过

**现有行为**：AI 返回 `category: "生活"` + 脱敏 summary → 仍创建 workRecord

**新行为**：
- AI 返回敏感分类（`生活` + summary 包含脱敏关键词）→ 跳过，不创建 workRecord
- 如果 autoDeleteScreenshots=true，同步删除截图
- emit 通知前端"已跳过 N 条敏感记录"

**敏感关键词**：私人沟通、社交媒体、浏览社交媒体、当前包含私人沟通、桌面空闲、查看桌面

### 4.3 隐私三档

| 级别 | 截图处理 | 敏感场景 | 适合人群 |
|------|---------|---------|---------|
| 宽松 | 保留截图 | 记录但脱敏 | 个人用户 |
| 标准 | 分析后删除 | 跳过不记录 | 大多数用户（默认） |
| 严格 | 分析后删除 + 不保存应用名 | 跳过 + 不记录 | 高度隐私敏感 |

**实现**：`privacyLevel` 字段为预设，切换档位时自动设置 `autoDeleteScreenshots` 和 `sensitiveSceneSkip`。用户也可在"截图与记录"卡片中单独微调（微调后 privacyLevel 显示为"自定义"）。

映射关系：
- `loose` → autoDeleteScreenshots=false, sensitiveSceneSkip=false
- `standard` → autoDeleteScreenshots=true, sensitiveSceneSkip=true
- `strict` → autoDeleteScreenshots=true, sensitiveSceneSkip=true（额外不保存 appName）

### 4.4 热力图时区 Bug 修复

**现象**：今天是 6 月 27 日，热力图只显示到 6 月 26 日。

**原因**：`electron/db.ts` 的 `heatmap()` 函数使用 `toISOString().slice(0, 10)` 提取日期，在 UTC+8 时区导致最后一天丢失。

**修复**：改用 `getFullYear()/getMonth()/getDate()` 本地方法（2 行代码），同时优化热力图样式（cell 尺寸、颜色渐变、hover 效果）。

**归入 Phase 1**：这是当前已有体验问题，应最先修复。

### 4.5 v1→v2 数据迁移

在 `electron/db.ts` 的 `getDb()` 中加入版本迁移逻辑：

```typescript
// 加载数据后检查 version
if (data.version < 2) {
  // 补模板 clustering 字段
  for (const t of data.templates) {
    if (!t.clustering) t.clustering = 'timeline'
  }
  // 补默认设置
  if (!data.settings.autoDeleteScreenshots) data.settings.autoDeleteScreenshots = 'true'
  if (!data.settings.sensitiveSceneSkip) data.settings.sensitiveSceneSkip = 'true'
  if (!data.settings.privacyLevel) data.settings.privacyLevel = 'standard'
  if (!data.settings.globalShortcut) data.settings.globalShortcut = 'Ctrl+Shift+J'
  if (!data.settings.subscription) data.settings.subscription = 'free'
  if (!data.settings.localApiEnabled) data.settings.localApiEnabled = 'false'
  if (!data.settings.localApiPort) data.settings.localApiPort = '8088'
  if (!data.settings.localApiToken) data.settings.localApiToken = ''
  data.version = 2
  save()
}
```

### 4.6 三种记录方式

| 方式 | 触发 | 状态 | source 字段 |
|------|------|------|------------|
| 定时自动 | 间隔 N 秒 | 已有 | `'auto'` |
| 全局快捷键 | Ctrl+Shift+J | 新增 | `'shortcut'` |
| 窗口内回车 | 非输入框时 Enter | 新增 | `'manual'` |

**全局快捷键实现**：
- `electron/main.ts`：`globalShortcut.register('Ctrl+Shift+J', () => captureNow())`
- 快捷键被占用时返回失败提示
- 可配置（`Settings.globalShortcut`）

**窗口内回车实现**：
- 渲染进程 `document.addEventListener('keydown', ...)`
- 排除 input/textarea/contentEditable 元素
- 调用 `window.api.screenshots.captureNow()`

### 5.4 报告模板增强

**聚类策略**：

| 模板 | clustering | AI 组织方式 |
|------|-----------|------------|
| 标准日报 | `timeline` | 按时间顺序排列 |
| 简洁日报 | `category` | 按分类归纳（开发/会议/文档…） |
| 技术日报 | `category` | 侧重代码开发和技术问题 |
| 项目日报 | `project` | 按项目维度组织 |
| 番茄钟聚类 | `timeline` | 按时间块聚类 |

**智能推荐**：
- 加载今日工作记录后，统计各 category 占比
- 占比最高的 category 映射到推荐模板：
  - 开发类 > 50% → 推荐"技术日报"
  - 会议/沟通类 > 40% → 推荐"简洁日报"
  - 多项目混合 → 推荐"项目日报"
  - 默认 → 推荐"标准日报"
- 推荐标签显示在模板卡片上："推荐 ✓"

**AI 提示词调整**（`electron/ai.ts`）：
- `timeline`：按时间线排列，标注每个时间段做了什么
- `category`：按工作分类归纳，每类下列出具体事项
- `project`：识别工作记录中的项目关键词，按项目分组

**报告生成流程 UI**：

```
点击"开始生成" → 按钮变为 loading + "正在生成..."
               → 下方展开流式渲染区（Markdown 实时渲染）
               → 完成后显示 [编辑] [导出 MD] [导出 TXT] [查看历史]
```

- 流式渲染区使用 `marked` 库渲染 Markdown
- 生成过程中按钮禁用，防止重复提交
- 完成后可直接在页面内编辑内容（内联编辑，不弹窗）

### 5.5 AI 洞察建议

**位置**：热力图和应用使用时长页面的数据下方

**实现**：
- 数据加载后自动调用 AI 生成一行简短洞察
- 流式显示，不阻塞页面渲染
- 仅在有数据时触发（空数据不调用 AI）
- **防抖 + 缓存**：同一页面同一数据范围只调一次 AI，结果缓存在组件内存中；用户手动点击"刷新洞察"才重新生成
- 用户可点击"展开详细分析"跳转 Agent 对话页继续深挖

**示例 prompt**：

```
以下是用户的工作热力图数据（日期 × 小时 × 记录数）：
{JSON 数据}
请用一句话总结用户的工作节奏特点和一个改进建议。不超过 100 字。
```

### 6.2 Agent 三层架构

Agent 功能分三层，覆盖不同用户群体：

#### 第 1 层：内置 AI 对话（覆盖 90% 用户）

**布局**：

```
┌────────────────────────────────────────────────────────────┐
│  AI 分析助手                                                │
│  基于你的工作数据，AI 帮你分析效率、发现模式、给出建议       │
├────────────────────────────────────────────────────────────┤
│  快捷提问：                                                  │
│  [分析本周工作模式] [找出时间黑洞] [效率提升建议] [总结亮点]  │
├────────────────────────────────────────────────────────────┤
│  对话区域（滚动）：                                          │
│  🤖 根据你本周的数据，你在"开发"上投入了 45% 的时间，        │
│     "会议"占了 30%。周三下午有明显的效率低谷...              │
│                                                              │
│  👤 能详细说说周三下午发生了什么吗？                          │
│                                                              │
│  🤖 周三 14:00-16:00 你切换了 12 次应用，主要在微信和        │
│     VS Code 之间来回。建议将沟通集中在固定时段处理...        │
├────────────────────────────────────────────────────────────┤
│  输入框 [发送]                                               │
└────────────────────────────────────────────────────────────┘
```

**实现**：
- 复用现有 AI 配置（baseUrl/apiKey/model）
- 对话时自动注入最近 7 天的工作记录 + 应用使用数据作为系统上下文
- 数据量控制：最多注入 200 条工作记录 + Top 20 应用时长，超出则按时间截断
- 快捷问题预设 prompt：

#### 第 3 层：本地 HTTP API（覆盖 1% 极客用户）

保留竞品的 HTTP API 能力，但做得更好。

**改进点**：

| 安全提示 | 无 | 设置页显示"本地 API 已开启"警告 |

**API 端点**：

Agent 接入配置文本：

```

**配置文本**（一键复制内容）：

```
请根据下面的说明把牙牙乐日报助手的 API 接入成你的 skill。

【服务信息】
本地 HTTP 服务地址：http://127.0.0.1:{port}
认证方式：Bearer Token: {token}

【强制规则】
1. 每次处理用户请求前，必须先调用 GET http://127.0.0.1:{port}/ 获取最新 API 文档
2. 读取文档后，解析接口列表，动态选择接口并构造请求

**实现**：
- 使用 Node.js `http.createServer()` 在主进程中启动

## 7. 竞品差异化优势

### 7.1 对比矩阵

| # | 维度 | 小黑日报 | 我们 |
|---|------|---------|------|
| 1 | Agent 能力 | 开放 API，用户自己对接 | 三层架构：内置 AI 对话 + 一键复制 + 本地 HTTP API |
| 2 | 数据洞察 | 只展示数据 | 数据 + AI 洞察建议 |
| 3 | 模板选择 | 30 种，选择困难 | 5 种精简 + 智能推荐 |
| 4 | 隐私控制 | 截图全删或全留 | 三档隐私级别 |
| 5 | 报告质量 | 只看工作记录 | 计划 vs 实际对比 |

### 7.2 竞品鸡肋分析

**Agent 能力**：小黑日报只开放本地 API 端口，用户需自己写代码对接 AI，普通用户根本用不了。我们的方案是三层架构：内置对话（零门槛）+ 一键复制上下文（粘贴给 Claude/GPT）+ 本地 HTTP API（极客用户，带认证和可配置端口）。

**30 种模板**：数量陷阱，很多模板只是标题不同，用户选择困难。我们精简到 5 种 + 智能推荐。

**隐私保护**：口号大于实质。截图要么全删要么全留，没有中间档。我们提供三档隐私级别。

**数据展示**：热力图/时长统计只展示数据，不给建议。我们在数据下方加 AI 洞察。
